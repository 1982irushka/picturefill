Picturefill A <picture> element polyfill

...with support for srcset, sizes, and type attributes too!

The picture element allows web developers to deliver an appropriate image to the user depending on a variety of conditions like screen size, viewport size, screen resolution, and more. Picturefill is a JavaScript file that enables support for the picture element in browsers that do not yet support it, so you can reliably use it today!

Basic markup pattern

Your picture element should contain a series of source elements followed by an img element. Each source element must have asrcset attribute specifying one or more image url sources (which can use expanded srcset syntax if desired for resolution switching). Additionally, you may add a media attribute containing CSS3 Media Queries, and/or a sizes attribute to pair with srcset.

Here's one with srcset and media. The first source that matches will determine the src of the img element at the end.

<picture>
	<source srcset="examples/images/extralarge.jpg" media="(min-width: 1000px)"></source>
	<source srcset="examples/images/large.jpg" media="(min-width: 800px)"></source>
	<source srcset="examples/images/medium.jpg"></source>
	<img alt="A giant stone face at The Bayon temple in Angkor Thom, Cambodia">
</picture>

IE9 Support

To support IE9, you will need to wrap a video element wrapper around the source elements in your picture tag. You can do this using conditional comments, like so:

<picture>
	<!--[if IE 9]><video style="display: none;"><![endif]-->
	<source srcset="examples/images/extralarge.jpg" media="(min-width: 1000px)"></source>
	<source srcset="examples/images/large.jpg" media="(min-width: 800px)"></source>
	<source srcset="examples/images/medium.jpg"></source>
	<!--[if IE 9]></video><![endif]-->
	<img alt="A giant stone face at The Bayon temple in Angkor Thom, Cambodia">
</picture>

Here's how that renders in the browser. Feel free to resize to see it change.

More examples

`media` and expanded `srcset` syntax:

<picture>
	<!--[if IE 9]><video style="display: none;"><![endif]-->
	<source srcset="examples/images/large.jpg, examples/images/extralarge.jpg 2x" media="(min-width: 800px)"></source>
	<source srcset="examples/images/small.jpg, examples/images/medium.jpg 2x"></source>
	<!--[if IE 9]></video><![endif]-->
	<img alt="A giant stone face at The Bayon temple in Angkor Thom, Cambodia">
</picture>

"sizes" attribute:

<picture>
	<!--[if IE 9]><video style="display: none;"><![endif]-->
	<source sizes="(max-width: 30em) 100%, (max-width: 50em) 75%, 50%"
		srcset="examples/images/pic-small.png 400w, examples/images/pic-medium.png 800w, examples/images/pic-large.png 1200w"></source>
	<!--[if IE 9]></video><![endif]-->
	<img alt="Obama with soldiers">
</picture>

`media` example with one webp source qualified with a type attribute.

Note: Picturefill supports SVG and WebP types on any source element, and will disregard a source if its type is not supported in a particular environment:

<picture>
	<!--[if IE 9]><video style="display: none;"><![endif]-->
	<source srcset="examples/images/large.webp" media="(min-width: 800px)" type="image/webp"></source>
	<source srcset="examples/images/large.jpg" media="(min-width: 800px)"></source>
	<source srcset="examples/images/medium.jpg" media="(min-width: 400px)"></source>
	<source srcset="examples/images/small.jpg"></source>
	<!--[if IE 9]></video><![endif]-->
	<img alt="A giant stone face at The Bayon temple in Angkor Thom, Cambodia">
</picture>

Inner img with fallback srcset attribute:

(NOTE: This pattern is probably not recommended due to a potential double-request in browsers that support srcset and not picture (a short-term problem, admittedly). See the first and second examples for a source-based catch-all fallback src.

<picture>
	<!--[if IE 9]><video style="display: none;"><![endif]-->
	<source media="(min-width: 9999px)" srcset="examples/images/pic-large.png"></source>
	<!--[if IE 9]></video><![endif]-->
	<!-- Fallback content. Note that using `srcset` will result in a duplicate request in browsers that have native support for resolution-only `srcset` but *not* `picture`. This is apt to be very few versions of evergreen browsers. -->
	<img srcset="examples/images/pic-small.png, examples/images/pic-medium.png 2x" alt="Obama with soldiers">
</picture>
